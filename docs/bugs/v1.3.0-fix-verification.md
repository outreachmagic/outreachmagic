## Remaining Item: Campaign UUID → Name Resolution

The Prosp campaign IDs are UUIDs (e.g. `43795293-6fcf-444b-b246-04671a947fcd`) — they're Prosp's internal sequence IDs, not the integer IDs in the local `campaigns` table. Three unique UUIDs across 1,541 events:

| UUID | Events |
|---|---|
| `43795293-6fcf-444b-b246-04671a947fcd` | 1,284 |
| `71381e3d-2a4f-4fbf-bda9-41f837d7d28d` | 196 |
| `98466961-c5aa-41ac-9b1a-500d5e99e688` | 61 |

These need to be mapped to human-readable campaign names for the stats pipeline. A mapping table or API call to Prosp's campaign endpoint would resolve them.

Calendly campaign_ids (3 UUIDs, 15 events) have the same issue but smaller volume.

## Bugs Fixed in This Commit

### Bug #8: `UPDATE_SCRIPT_FILES` whitelist blocks new `.py` files

**Severity:** MEDIUM — blocks future script additions via `pipeline.py update`

**Root cause:**
```python
UPDATE_SCRIPT_FILES = tuple(
    sorted(p.name for p in Path(__file__).resolve().parent.glob("*.py"))
)
```

This computes the file list from the **currently installed** scripts directory. When `update_download_names` filters the manifest by this list, any new `.py` file (like `reprocess.py`) is excluded because it doesn't exist locally yet. This is a bootstrapping/catch-22 problem.

**Fix:** Changed `update_download_names()` to use `.py` extension matching instead of the local file whitelist. Any `.py` file in the remote manifest is now installed regardless of whether it existed in the previous version.

### Bugs #9 & #10: `reprocess.py` runtime errors (fixed in test suite updates)

Both bugs were discovered during reprocess testing and are fixed in the staged changes:
- **#9:** Added `import sys` so `--verbose` flag works
- **#10:** Removed `updated_at = datetime('now')` from UPDATE SQL — the `events` table doesn't have that column
